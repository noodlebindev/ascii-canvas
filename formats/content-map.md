# content-map

## Purpose

Take one core idea and expand it into derived content artifacts — hooks, posts, threads, angles.

## Use when

- Social content planning: one idea → many posts
- X / LinkedIn content strategy from a single insight
- Mapping angles, hooks, and formats from a core message
- "Give me a content plan for X" requests

## Do NOT use when

- Ideation / brainstorming a topic — use `mind-map`
- Concept dependency or layered understanding — use `concept-stack`

## Width budget

100 cols

## Arc

A6 — Content-map arc (3–5 clusters): core idea → angle clusters → optional distribution

## Layout rules

- Core idea at top in heavy `╔═╗` border spanning full width
- 3–5 angle-cluster boxes in a horizontal row below
- Each cluster contains 2–4 content units with `▸` prefix (Hook, Post, Thread)
- Optional distribution notes in a single line at bottom
- Equal-width cluster columns

## Failure modes

- If clusters exceed 5 → group or prioritize to 3 strongest angles
- If each cluster has only 1 content unit → fall back to `mind-map`
- If width overflows 100 cols → shorten hook/post/thread text or reduce to 2 clusters
