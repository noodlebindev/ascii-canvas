# mind-map

## Purpose

Central idea with radial branches of related concepts.

## Use when

- Brainstorming вҖ” exploring all facets of a topic
- Ideation around a core concept before writing or building
- Mapping related ideas that don't have a strict hierarchy

## Do NOT use when

- Content packaging across multiple formats вҖ” use `content-map`
- Strict dependency hierarchy вҖ” use `concept-stack`

## Width budget

100 cols

## Arc

N/A вҖ” single artifact

## Layout rules

- Central `[Core Idea]` box at top (or left-center for wide layouts)
- Branches spread below with `в”¬` / `в”ң` / `в””` connectors
- Depth вүӨ 3: root вҶ’ branch вҶ’ leaf
- Each leaf is one concept, formatted as `вҖў <concept>`
- No crossing lines

## Failure modes

- If depth exceeds 3 вҶ’ switch to `concept-stack` or `content-map`
- If layout is too wide вҶ’ reorient to vertical (branch list with indented sub-bullets)
- If branches need ordering or dependencies вҶ’ switch to `concept-stack`
